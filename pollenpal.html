PollenPal

Overview

PollenPal is an allergy management app designed to empower users to track and minimize their exposure to seasonal allergies. By enabling users to anticipate high-risk days and take preventive action, PollenPal helps them stay ahead of allergy symptoms.The objective is to give users timely, location-based pollen forecasts to help them plan their day and reduce exposure to allergy triggers. Allergy sufferers can be caught unprepared by changing pollen levels. Our app solves that with real-time forecasts and accessible, visual insights.

This is a two person school project, and was inspired by my partner's personal experiences with seasonal allergies. She found herself frequently affected by unexpected changes in pollen levels, and we realized there were not many mobile tools designed specifically to help allergy sufferers manage their symptoms in real-time. That need formed the foundation for PollenPal's purpose and feature set.

First Iteration

In the initial phase of development, we focused on creating a prototype that shows how core features such as location-based pollen forecasts and pollen database would be displayed. The first iteration allowed us to visualize our concept and make sure we are on the same page.

Problems

The biggest problem with this iteration is that the response that are recieved from the API was not as precise as we thought. For example, while having separate pollen levels in each region for specific plants was sometimes available, it was inconsistent and not viable for us to implement, especially with our time constraints.

One of the biggest changes to the interface will be the way in which pollen levels are displayed. The levels of pollen were not given in a quantitative measurement like grains per cubic metre, they are measured on a scale of 0-5: 0 being little to no pollen in the air, and 5 being extremely high pollen levels. So, the specific measurements are omitted, and the line graph is redundant, making the interface a lot more simple.

Development Process

The main bulk of this project was working with the structured data from Google's APIs and the implementation of Firebase. I wrote logic to extract the relevant pieces of information from the Pollen API and the Google Maps API, convert them into plain, human-readable content, and then connect them to the visual components of the app. I also supplemented that with functions to help generate additional insight into the information retrieved from the API. Alongside this, implementing Firebase Authentication and database allows users to create and manage accounts, as well as access a database of in depth information on regional pollinator plants. I also supported frontend integration by helping wire backend logic to the interface my partner developed.

Challenges

The most challenging part of the project was coming up with a way to parse the data retrieved from the API. It took a lot of trial and error to write code that will properly extract the needed data from the api response and converting it to something that the app can display without any errors. As this was my first mobile app project, I had to learn React Native, Firebase integration, and API workflows from scratch. The learning curve was steep, but incredibly rewarding.

Future Prospects

If we had more time, I think I would have liked to add push notifications and alerts, similar to a daily alarm or a daily weather update in the morning. In terms of the less functional aspects, I think implementing some sound effects throughout the app and to accompany the Pal to make it feel more personable and alive. In the same vein, adding the ability to customize the pal could add a playful aspect to the app.

Summary

We delivered a fully functional prototype of PollenPal. It provides users with real-time pollen forecasts based on their location in a clean, intuitive interface. The app is reliable and lays the foundation for future features like push notifications and advanced personalization. From a personal perspective, this project gave me practical mobile application development experience. I learned a lot and improved my skills in backend development, data handling, mobile UI logic, and working closely with a teammate in a collaborative environment.
